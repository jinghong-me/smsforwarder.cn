隐私政策

最后更新时间：2026年4月6日

概述

短信转发助手（以下简称"我们"）非常重视用户的隐私保护。本隐私政策说明了我们如何收集、使用、存储和保护您的个人信息。使用我们的应用即表示您同意本政策中描述的做法。

信息收集与处理

短信内容及相关信息（敏感个人信息）

类型：收到的短信文本内容、发送者号码（原始号码）、短信时间戳等。

用途：用于在设备上匹配用户配置的关键词规则并执行转发（例如发送到用户配置的 webhook 或通过设备发送到目标手机号）；同时记录本地转发日志以供用户查看和排查。

上传说明：应用不会将短信内容或相关敏感信息自动上传到开发者服务器或任何第三方服务器。唯一会把短信内容发送到网络的情形是：用户自行在应用中配置了一个 webhook（HTTP/HTTPS URL）作为转发目标；应用会将短信内容直接 POST 到该用户指定的第三方 URL（请在配置时谨慎选择接收方并确认其隐私做法）。开发者不对这些第三方接收方的数据处理负责。

我们仅在您明确授权的情况下访问短信内容，用于转发功能。我们不会存储您的短信内容，仅在转发过程中临时处理。

发送短信相关信息（仅在用户选择 SMS 通道时）

类型：用户为"SMS"类型通道配置的目标手机号、（可选）用于发送的 SIM subscriptionId（subId）。

用途：当用户在通道管理中添加 SMS 类型通道且启用该通道时，应用会通过设备的 SmsManager 使用设备的短信功能将原始短信内容转发到用户指定的目标手机号。

上传说明：发送操作仅在用户设备本地通过系统短信服务完成；应用不会将发送的短信内容发送至开发者服务器。

设备与运行信息（非敏感 / 技术性）

类型：应用版本、Android 版本、设备型号、运行时日志（应用内部日志文件）、崩溃或错误信息（默认不自动上报）。

用途：用于本地诊断、日志记录和用户在应用内查看转发历史/调试信息。

上传说明：本应用不集成任何自动上报的第三方崩溃分析或统计 SDK；也不会默认将日志/崩溃信息上传到开发者服务器。

应用请求的权限与用途

RECEIVE_SMS / READ_SMS（接收/读取短信）

用途：监听并读取设备收到的短信内容，用于在设备端按用户配置进行转发与规则匹配。若用户拒绝该权限，应用无法读取到短信，核心转发功能将无法使用。

SEND_SMS（发送短信）

用途：当且仅当用户在"转发通道"中添加并启用类型为 SMS 的通道时，应用会使用系统短信功能将短信内容转发到用户配置的目标手机号。若用户不添加 SMS 通道或未授予该权限，应用不会发送短信。

READ_PHONE_STATE（用于读取 SIM / subscription 信息）

用途：在用户选择 SMS 通道并需要指定发送 SIM 卡（多卡设备）时，用于读取卡信息并允许用户选择要用于发送的 subscriptionId（subId）。此权限为按需请求，仅在需显示/选择卡信息时请求；若未授权，应用仍可使用默认短信管理器发送（但无法列出/选择具体 SIM 卡）。

RECEIVE_BOOT_COMPLETED（开机自启）

用途：当用户在设置中启用"开机自启动"后，应用会在设备开机完成后启动前台服务以保持转发服务持续运行；若用户不启用或撤销此权限，应用不会自动在开机时启动。

POST_NOTIFICATIONS（通知权限）

用途：用于显示应用的前台服务常驻通知（前台服务要求），提示服务状态与快速操作（例如停止服务）。应用在启动服务前会检查并提示用户开启通知权限。

INTERNET（网络访问）

用途：用于将短信转发到用户配置的网络通道（Webhook 等）。仅在用户主动配置 Webhook 时用于网络请求；默认不上传任何数据至开发者服务器。

数据存储与备份

服务器存储

本应用当前不会将短信、联系人或转发记录上传到开发者服务器或第三方服务器（开发者也未集成任何收集类 SDK）。

本地存储

您的个人信息和短信内容应用在设备内部存储（context.filesDir）保存。转发日志文件（文件名：sms_forwarder_logs.txt），用于在应用内展示最近的运行/转发信息；该文件保留最近若干条（源码中限制为最多 200 条）；应用配置与通道/关键词规则保存在 SharedPreferences（名称 "app_config"）中。

设备备份

应用在 Manifest 中允许备份（android:allowBackup="true"），这意味着在设备或用户启用的系统备份机制下，应用数据可能被包含在系统备份中；如需避免被备份，请在设备/账户备份设置中管理。

用户权利与可控性

权限控制

用户可通过 Android 系统权限管理随时撤销或授予权限（例如 RECEIVE_SMS、SEND_SMS、READ_PHONE_STATE、POST_NOTIFICATIONS 等）。撤销权限会影响对应功能。

数据查看与删除

本地日志可以在应用内查看、刷新或清除（应用 UI 提供清除日志功能）。应用数据（配置、通道、关键词）保存在本地 SharedPreferences，用户可以在应用内删除或通过卸载应用清除。

如果确有场景导致开发者持有用户数据（当前版本通常不会），用户可通过联系邮箱 huahao@email.cn 提出数据访问或删除请求。开发者在验证请求人身份后将在合理期限内处理（通常不超过 30 天）。

导出

若需要导出本地数据（例如日志或配置），用户可在设备上手动导出或开发者将根据具体功能在应用内提供导出入口（当前版本未提供自动上传/导出到开发者服务器的行为）。

数据共享与第三方

重要声明：我们不会采集、共享、出售您的个人信息给第三方。

用户指定的转发目标（Webhook、第三方服务或目标手机号）

当用户将转发目标设置为第三方 URL 或服务时，应用会把短信内容直接发送到该第三方目标（通过 HTTP POST 或发送 SMS），这些数据会由该第三方接收并由其处理。开发者不会默认收集或转发这些数据到自己的服务器，且不对第三方的隐私实践负责。用户应谨慎选择转发目标并确认其隐私与安全政策。

第三方 SDK

当前版本未集成任何第三方数据收集类 SDK（例如统计、崩溃、广告等）。如未来集成会在隐私政策中说明并征得必要同意。

政策更新

我们可能会不时更新本隐私政策。重大变更时，我们会通过应用内通知或其他方式告知您。建议您定期查看本政策以了解最新信息。
